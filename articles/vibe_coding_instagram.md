# Vibe Coding · Réflexion

Le potentiel est prouvé. Et si les vraies limites venaient de **l'écosystème** lui-même ?

---

## La promesse est réelle

On est passé de *"l'IA complète mes lignes"* à *"l'IA génère un module entier"*.

**Claude Code. Cursor. Copilot. Devin.**

Les démos sont bluffantes. Les progrès, spectaculaires. Et pourtant — ceux qui pratiquent au quotidien savent : **on se heurte à des plafonds**.

---

## Les plafonds que personne ne montre

- **Boucles de correction infinies** — l'agent se corrige sans jamais converger
- **Hallucinations sur les dépendances** — la librairie n'existe tout simplement pas
- **Code opaque** — généré, mais impossible à faire évoluer
- **Agents qui perdent le fil** dès qu'on sort d'un seul fichier

On accuse les modèles. *"Pas encore assez bons."* C'est en partie vrai. Mais je me demande si on ne regarde pas au mauvais endroit.

---

## Et si le problème venait de l'écosystème ?

> On demande à des LLM de travailler avec des outils optimisés pendant **70 ans pour les humains**.

**Python** est lisible — *pour un humain*.
**Django** évite la réécriture — *à un humain*.
**Git** structure la collaboration — *entre humains*.

Tout a été pensé pour nos limites cognitives. **Pas pour des agents IA.**

---

## Les coutures qui craquent

Un LLM dépense des **tokens** à écrire des docstrings que personne ne lira.

Un agent suit les conventions d'un framework dont il **n'a pas besoin**.

Deux agents collaborent via Git alors qu'ils pourraient **échanger des représentations structurées directement**.

On vérifie du code généré avec des tests écrits par un autre LLM — *du probabiliste sur du probabiliste*.

---

## 4 Paradigmes pour la prochaine étape

- **Représentations intermédiaires** pensées pour la génération machine — concises et vérifiables
- **Vérification formelle intégrée** — l'IA génère, un prouveur certifie
- **Protocoles inter-agents** avec négociation de capacités et gestion d'incertitude
- **Versionnement sémantique** qui trace les *intentions métier*, pas les diffs textuels

On n'a pas besoin de tout réinventer d'un coup. Mais **étape par étape**, on peut adapter les fondations.

---

## Générer du code — c'est acquis.

Le prochain palier :

**Concevoir une architecture. Vérifier la qualité. Orchestrer un déploiement.**

Il ne sera atteignable que si **l'écosystème suit**.

Le Vibe Coding a prouvé le potentiel. Maintenant, il faut **adapter les fondations**.
